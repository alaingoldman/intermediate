Review

HTML Tags

HTML stands for Hyper Text Markup Language. It's the language of web browsers. HTML is composed of a series of tags. Each tag is a keyword inside of an angle brace. This tag is also known as a start tag.

Most tags need a closing tag as well. Which is the same as the start tag with a forward slash infront of the keyword. If you are interested you can see an expansive list of HTML tags here.


    <p class="my-style">
      Hello World!
    </p>

As you can see in this HTML code. We are using the p tag with "Hello World!" inside of it. Everything from the start tag to the closing tag is considered a HTML element.

You can also have an attribute, which adds information to the HTML element. Attributes are only defined in the start tag. They are set in name/value pairs. For example name="value". In the above example the p tag has a class attribute set to the value of "my-style" .

The most popular attributes are class and id. You can set values for the class or id attribute as whatever you want but your HTML should only contain one type of that id. Where you can have multiple classes of that type on the page.

HTML Structure


All HTML files should contain this structure.


   <!DOCTYPE html>
   <html>
     <head>
       <title> My First Page </title>
     </head>
     <body>
       I hope you like it.
     </body>
   </html>

Each HTML file should contain these:

  • Doctype : Tells the browser which verison of HTML to use.
  • HTML : The HTML tag denotes the start and end of the webpage.
  • Head : The head tag contains site information; styles, scripts and meta info.
  • Body : Contains the body of the document and is displayed in the browser.

Links & Images


    <img src="logo.png" width="40">
    <a href="index.html">
      Click Here
    </a>

The img tag is how to add images to your site. This tag does not need a closing tag. The src attribute points to the location of the image. This can be a local image or it could be a link to an image file online.

The img tag can be supplied a width or height attribute and this will shape the image to your restrictions in pixels. Note that if you supply only one of the size attributes the images ratio will not be destroyed. However, if you add both size attributes the image ratio maybe destoryed.


If you want to create a link to another page, be it on your site or another site, you need to use the a tag. Which is also called the anchor tag. Use the href attribute to denote the target file. The content inside of the a tag will be represented as a link. It could be text, image or even any number of HTML elements.

Targeting with Selectors



    selector {
        property:value;
    }

In CSS you use a selector to target the elements you want to apply styles to. One way you can select and element is by looking for it's id or class attribute. If you want to select an element with the id value of content and another with the class value of wrap you would write this : 


                    
    #content {
      font-family: 'Lato', arial, sans-serif;
      font-size: 16px;
    }
    .wrap {
      margin: 0 auto;
    }

Here is a list of some other ways to target elements.


Select all elements on the page with * 

    *{margin:0;padding:0;}
Select based on the type of tag. 

    a {text-decoration:none;}
    body {background-color:red;}
Select descendents. 

    li a{
      text-decoration:none;
    }
You can join multiple selectors with a comma.
(Yes, that is a comma between .top and .bottom.) 

    .top ,.bottom{
      margin:10px;
    }

Media Queries


    @media screen and (max-width: 700px) {
        header {
            background-color: #343434;

The @media rule is used to define different style rules for different media types/devices. This can check things like width, height, resolution and orientation. Here I set that if the width of the browser is anything under 700 pixels change the background-color: of the header to #343434;.

Positioning

Block vs Inline


A block element is an element that takes up the full width available, and has a line break before and after it.

Examples of block elements:
  • <h1>
  • <p>
  • <li>
  • <div>

An inline element only takes up as much width as necessary, and does not force line breaks.

Examples of inline elements:
  • <span>
  • <a>

Alternatively there is also an inline-block display. Meaning this element is displayed as a block but is formatted as an inline element.

Horizontal Centering (block elements)

If you want to center a block element on a page horizontally you can do it one of two ways. Either with margin or absolute positioning.

Margin 

    <div class="box"></div>
                    

    .box{
      height:100px;
      width:100px;
      background-color:blue;

      margin-left:auto;
      margin-right:auto;
      /*margin:0 auto;*/
      /*margin:0 auto 0 auto */
    }

This specifies that the element should split the available margin equally. There are other shorthand ways of setting the margin-left and margin-right. Both of which I commented out in the CSS above.


Absolute Positioning 

    <div class="box"></div>
                    

    .box{
      height:100px;
      width:100px;
      background-color:blue;

      position:absolute;
      left:50%;
      transform:translateX(-50%);
      -moz-transform:translateX(-50%);
      -webkit-transform:translateX(-50%);
    }








After applying position : absolute ; you can use left , right , bottom and top to move the div around accordingly. On a couple of CSS properties you will see "-moz-" or "-webkit", these prefixes will just make sure the change happens on firefox and chrome browsers respectively. The transform property was added in CSS3 and only works in newer browsers. If you want to support older browsers you will have to do this another way.


    .box{
      height:100px;
      width:100px;
      background-color:blue;

      position:absolute;
      left:50%;
      margin-left:-50px;
    }

What I want you to notice is that in order to center this div with absolute positioning perfectly, you need to know the width of the element. I set the margin-left equal to minus half the width of the div in order to shift it over because absolute positioning will start the div at exactly half the size of the container without taking into account the size of the element itself.
So, if you do not know the width of the element you want to center this is not a good option.

Another issue with aboslute positioning is that the div doesn't follow the same flow of the page and because of that it does not take up space the same way other elements do. This in turn will cause overlapping of elements. In order to show this example in this website itself I had to add a series of break tags after the div to forcefully give a space that the div would land on.

Cons
  • Width has to be known.
  • Element doesn't follow flow of the page.

With all this being said you should only use absolute positioning in extreme cases.

Fixed Elements

Earlier we briefly touched absolute positioned elements. This is when the element is positioned relative to its first positioned (not static) ancestor element. This element then does not follow the flow of the webpage. This is an interesting contrast to position : fixed ; where the element is positioned relative to the browser window. An example of this is the nagivatgion panel on this website. No matter how far the user scrolls down the navigation panel is still there.

Here is an example of how you could create a fixed element that covers the whole page. 


    <div class="annoying-box"></div>
    <button id="bother">Warning! Show the annoying box<button> 
                    

    .annoying-box{
      height: 100%;
      width: 100%;
      background-color: orange;

      position: fixed;
      left: 0;
      top: 0;

      display:none; /* hides it */
  }
                    

    $('#bother').click(function() {
      $('.annoying-box').show();
    });
    $('.annoying-box').click(function() {
      $('.annoying-box').hide();
    });



I provided the jQuery on how I show / hide this element if you're interested.


Float

Floating elements is a seemingly simple concept but it is one of the most confusing things in front end development. When you float an element it becomes a block box. This box can then be shifted to the left or right on the current line.


    <div class="red"></div>
    <div class="blue"></div>
                    

     div{ height:100px; width:100px; }
    .red{
      background-color:red
      float:right;
    }
    .blue{
      background-color:blue
      float:left;
     }







What I want you to notice is that I make the red div first and the blue div second but the positions are switched in the browser because they are floated.


Clearfix - A CSS Hack

Sometimes in older browsers a floating element may escape it's container. As you can see in the last example both the blue and red div's are contained within the white content wrapper of this website and are working correctly. In older browsers however this would require a hack to get around. We will not be going over this hack in depth but you can see it here if you ever need to implement it. It's a problem web developers used to have but we shouldn't have as much anymore.

More CSS

Cascading


There can be a number of styles in a HTML document but by the time it is rendered by your browser all of the stylesheets a cascaded into one. So if one element has two CSS rules, and the rules have the same weight, the latter one wins.


        <div></div>
                        

        div{
          height:100px;
          width:100px;
          margin:0 auto;
          background-color:red;
        }
        div{
          background-color:blue;
        }

There is an order of priority which won't really an issue for us in our examples today. I just want you to know that an order exists. If you want addition resources on that you can check it out here.

However, if you want something to override regardless of it's position in the cascase you can use !important . This should only be used in extreme cases. Only because if anther developer adds changes to the same element after you did no matter what your !important wins. Which will cause issues down the line. You write !important before the semicolon.


        <div></div>
                        

        div{
          height:100px;
          width:100px;
          margin:0 auto;
          background-color:red !important;
        }
        div{
          background-color:blue;
        }

Psuedo Classes


A Pseduo class is used to define a special state of an element. For example when a element is hovered over or when an a tag is visited. 

    selector:pseudo-class {
        property:value;
    }

This is how it works. Here are some examples of a couple of Pseudo classes and how to use them.


: hover 

    .box {
        background-color:black;
        height:100px;
        width:100px;
        margin:0 auto;
    }
    .box:hover{
        background-color:pink;
    }




: nth-of-type(n) with :: before + :: after 

    #dogs li:nth-of-type(3)::before {
        content:"Bad ";
        color:red;
    }
    #dogs li:nth-of-type(3)::after {
        content:" !";
    }
                        

    <ul id="dogs">
        <li>Dog</li>
        <li>Dog</li>
        <li>Dog</li>
        <li>Dog</li>
    </ul>
  • Dog
  • Dog
  • Dog
  • Dog

This CSS is a bit complex but it says that target the id dogs. Then target an li element inside of it. Then apply this property change to the 3rd li within the dogs id. Then with ::before add before the element the content "bad " and make the color of the text red. Then with ::after add the content of " !".

If you want to see a more expanisive list of pseudo elements you can check them out here.

Opacity

There are two ways to go about doing opacity.


    <div> 
      <h2>
        I can barely see this
      </h2>
    </div>
                        

    div{
        height:38px;
        background-color:black;
        opacity:0.1;
        text-align:center;
    }

I can barely see this

(hightlight this div to see the text)


Setting opacity: 1; is 100% so opacity is represented with decimals. Now this works but the opacity of the element effects all the opacity of the children.

To change the opacity of a div and not it's children you would have to do something like this:


    <div> 
      <h2>
        I can see this
      </h2>
    </div>
                        

    div{
        height:38px;
        background-color:rgba(0,0,0,0.1);
        text-align:center;
    }

I can see this




You would do this if you have to lay text over an image

CSS Animations

@Keyframes

CSS gives the ability to render animations without jQuery or flash. 


div {
  width: 100px;
  height: 100px;
  background-color: red;
  position: relative; 

  /* Chrome, Safari, Opera */
  -webkit-animation-name: example;
  -webkit-animation-duration: 5s;
  -webkit-animation-timing-function: linear;
  -webkit-animation-delay: 2s;
  -webkit-animation-iteration-count: infinite;
  -webkit-animation-direction: alternate;

  /* Standard syntax */
  animation-name: example;
  animation-duration: 5s;
  animation-timing-function: linear;
  animation-delay: 2s;
  animation-iteration-count: infinite;
  animation-direction: alternate;
}

/* Chrome, Safari, Opera */
@-webkit-keyframes example {
  0%   {background-color:red;    left:0px;  top:0px;}
  25%  {background-color:yellow; left:85%;  top:0px;}
  50%  {background-color:blue;   left:85%;  top:200px;}
  75%  {background-color:green;  left:0px;  top:200px;}
  100% {background-color:red;    left:0px;  top:0px;}
}

/* Standard syntax */
@keyframes example {
  0%   {background-color:red;    left:0px;  top:0px;}
  25%  {background-color:yellow; left:85%;  top:0px;}
  50%  {background-color:blue;   left:85%;  top:200px;}
  75%  {background-color:green;  left:0px;  top:200px;}
  100% {background-color:red;    left:0px;  top:0px;}
}
